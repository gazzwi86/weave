## Decisions Made

*(reconstructed by QA from commit `d24ec75`'s body and `docs/specs/weave/engines/build-engine/decisions/ADR-010.md` — Engineer should confirm this is complete)*

- **Authz falls back to tenant-admin** (`rbac.require_tenant_admin`), not a separate
  company-admin/project-admin role — ADR-010, because no such roles exist in M1 identity. Flagged
  in the ADR for human ratification. QA assessed this as a reasonable fail-closed equivalent, not
  a regression (see QA verdict).
- **Two partial unique indexes**, not one plain `UNIQUE`, on `standards_documents` — Postgres
  treats `NULL` as distinct-from-itself, so a single constraint would let unlimited company-scope
  duplicates through (`project_id` always `NULL` there).
- **`StandardsConflictError` carried as data, not raised** — a cross-key `stack_pins` conflict
  degrades only the conflicting axis to demo-default; it must not halt the other, non-conflicting
  axes or generation as a whole.
- **`effective_set` kept as a pure, DB-free function** (task's implementation hint) — the repo
  layer (`load_effective_standards`) fetches the two `active`-status lists separately.
