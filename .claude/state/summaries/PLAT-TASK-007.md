  Slack attempt on every path. The full preference schema, delivery pipeline, and retry/backoff

  template for any future external-connector stub (Law F — no live SaaS in tests).

- **Badge is fetch-on-mount/fetch-on-open, no SSE/polling** — deliberate ponytail cut (noted in
  `use-notifications.ts`). The brief's hints suggested SSE; no AC requires it. Follow-up task if
  billing/audit need near-real-time badges.
- **NotificationList split out of notification-center.tsx** — original component was 74 lines,
  over Law E's 50-line budget.

## Nuances

- **Global light-theme token bug found by the new axe test and fixed at the root**
  (commit 77e12f0): `--color-success/-warn/-danger/-info` had NO light-theme overrides in
  `app/globals.css` (unlike `--color-accent-*`) — dark hex leaked into light mode, e.g.
  `--color-info` at 1.66:1 on white (WCAG 1.4.3 fail). Post-fix contrast recomputed by QA:
  5.48/5.02/6.47/5.93, all ≥4.5:1 AA. Affects every component using those tokens — ledger row
  filed for traceability; broader design-system audit worth considering.
- **New hard E2E dependency**: `NotificationCenter` mounts globally in `AppShell`, so EVERY page
  now needs a live, migrated Postgres for E2E (`docker compose up -d postgres redis` +
  `uv run python -m weave_backend.db.migrate`). Previously only DB-touching specs needed docker.
  CI already migrates; local-dev friction flagged.
- QA closed a real coverage gap: `mark_read_route` and `update_preferences_route` had zero
  HTTP-level integration coverage before this pass (unit-tested against fakes only).
- Mutation testing not run locally (mutmut not in venv) — WARN not block; mutation is CI-owned
  per PROJ-005/ADV-003.
