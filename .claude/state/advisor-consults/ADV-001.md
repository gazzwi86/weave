## Required-before-merge (blocking) — dispositions

1. **False backstop claim + force bypasses** — the "blocked server-side by branch protection" claim
   is false (private repo, no protected branches; `gh api` returns 403). RESOLVED: removed the claim
   from `git_safety.py` + `git-safety.md` (hook is sole enforcement); closed bypasses — `+refspec`
   force syntax, quoted `"--force"`, `-f` in a neighbouring command (scoped to the push segment),
   `--force-with-lease` with no refspec resolved against HEAD, `main` inside hyphenated branch names.
   Test grown 10→18 cases (all pass).
2. **`restack.sh` dies on macOS bash 3.2** (`declare -A`, `mapfile`). RESOLVED: rewritten with
   indexed arrays + C-style loop; `bash -n` verified on 3.2.57.
3. **Corrupting conflict-recovery path.** RESOLVED: old tips captured up front; on conflict the
   script prints the exact `git rebase --onto <parent> <old-tip> <child>` commands for every
   remaining branch instead of an unsafe bare re-run.
4. **Discovery includes merged branches.** RESOLVED: discovery now uses open PRs
   (`gh pr list --state open`) and skips branches already merged into `origin/main`.
5. **Repo merge settings contradict git-workflow.md §6** (squash enabled, merge-commit disabled).
   DEFERRED to human — changing repo settings is outside the agent's autonomy; one-liner handed to
   the operator (`gh repo edit --enable-squash-merge=false --enable-merge-commit=true
   --enable-rebase-merge=true`). Logged in the QA ledger.
