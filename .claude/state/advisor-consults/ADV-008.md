## Rationale (advisor's summary)

The two-pass grep strategy (literal-ID pattern, then broader bare-word pattern) is the right
completeness check for a rename sweep like this — literal IDs alone miss prose that names a model by
short name without the full ID. The one real defect found (`testing-strategy.md`) was a partial
conversion that changed a pseudocode function's return shape without updating the surrounding
contract language describing that shape — a common failure mode when a mechanical find-replace
gets applied unevenly across a schema and its call sites. Reverting to the literal-ID exception
(rather than carrying the tier rename through `ALLOWED_MODELS` and both `ModelRoutingError`
phrasings) was the lower-blast-radius fix. No other file in the diff has a sibling half-conversion
(confirmed via `grep -rn resolve_model docs/specs/weave` across the non-frozen tree). The
`harness-governance.md` and `stop.py` enforcement-core edits are both benign (factual sync of
`advisorModel`'s current value; a comment generalized from an enumerated model list to "base-tier
model") — neither weakens a gate, so harness-governance.md rule 4 is not implicated.
