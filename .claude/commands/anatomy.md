# /anatomy ‚Äî repository anatomy generator

You maintain the project's **anatomy map** ‚Äî a navigable, semantic index of the codebase that
lets an agent traverse one or two files instead of grepping the whole tree.

The map has three layers:

- **`ANATOMY.md`** (repo root) ‚Äî top-level table: one row per area, one-sentence purpose,
  link to its wiki page.
- **`docs/wiki/<area>.md`** ‚Äî one OKF concept per area: key symbols per file, semantic
  description of each. This is the bundle body.
- **`docs/viz.html`** (gitignored) ‚Äî self-contained Cytoscape.js graph rendered from the
  unified `docs/` bundle (wiki + specs + standards); regenerated by `/anatomy` on full runs.

`docs/wiki/` is an **OKF v0.1 bundle**. Every non-reserved `.md` file must have YAML
frontmatter with at minimum a `type` field. See the OKF spec summary at the bottom of
this file.

`<area>` is derived from a file's path exactly as `.claude/scripts/modules/common.py::area_for_path`
does it (e.g. `apps/web/...` ‚Üí `web`, `packages/core/...` ‚Üí `core`, `src/api/...` ‚Üí `src-api`,
`infra/...` ‚Üí `infra-<sub>`). Only files under the tracked roots `apps/ packages/ infra/ src/ lib/`
have wiki pages ‚Äî everything else (docs, prototypes, .claude config) is intentionally excluded.

## Arguments

`$ARGUMENTS`

- **`refresh <file1> <file2> ‚Ä¶`** ‚Äî incremental mode (called by the pre-commit / pre-push hooks).
  Only the areas containing those files are touched. Does **not** re-render `viz.html`.
- **no arguments** ‚Äî full regeneration of `ANATOMY.md`, every `docs/wiki/<area>.md`, the bundle
  `index.md` / `log.md`, and `viz.html`.

## Procedure

### Step 1 ‚Äî Determine target areas

- **refresh mode**: map each passed file to its area via the rule above. Skip files not under a
  tracked root. If no files map to an area, **make no changes and exit**.
- **full mode**: discover all areas under the tracked roots (`Glob` `apps/**`, `packages/**`,
  `src/**`, `lib/**`, `infra/**`).

### Step 2 ‚Äî Gather structure per area

Prefer jcodemunch when the repo is indexed (`resolve_repo` ‚Üí if `indexed`, use `get_file_outline` /
`search_symbols` for accurate symbol lists). Otherwise read the files directly.

Capture per file: its purpose, and each exported/public symbol with a ‚â§1-line description.
Identify 2‚Äì4 relevant tags for each area (e.g. tech-stack layer, framework, domain).

### Step 3 ‚Äî Write `docs/wiki/<area>.md` (OKF concept)

Each area page is an OKF concept. Write it with this exact shape:

```markdown
---
type: Code Area
title: <area> ‚Äî <one-sentence purpose>
description: <one-sentence description, same as title suffix>
tags: [<tag1>, <tag2>]
timestamp: <UTC datetime in ISO 8601, e.g. 2026-06-29T12:00:00Z>
---

_Anatomy ‚Äî regenerated by `/anatomy`. Last refresh: <UTC date>._

## <relative/file/path>
<one-line file purpose>
- `symbolName(args) -> ret` ‚Äî what it does
- `ClassName` ‚Äî responsibility
```

Rules:
- The YAML frontmatter block **must** come first, before any other content.
- Stale markers (`<!-- stale: ... -->`) appended by the hook sit at the bottom of the file,
  **after** all content. When refreshing, remove every `<!-- stale: <path> -->` marker for files
  you just re-indexed; the hook re-adds them the next time those files change.
- Cross-link to related area pages using bundle-relative markdown links:
  `[web](web.md)` or `[src-api](src-api.md)`.

### Step 4 ‚Äî Update `docs/wiki/index.md` (full mode only)

`index.md` is an OKF reserved file (¬ß6). The bundle-root `index.md` may carry only
`okf_version` in its frontmatter. Write it as:

```markdown
---
okf_version: "0.1"
---

# Weave ‚Äî Code Knowledge Bundle

Auto-generated by `/anatomy`. One concept per code area. Open a page before grepping.

## Areas

| Area | Purpose |
|---|---|
| [web](web.md) | Next.js SPA ‚Äî graph explorer + spec UI |
```

In refresh mode, update only the rows for touched areas (add rows for new areas).

### Step 5 ‚Äî Update `docs/wiki/log.md` (full mode only)

`log.md` is an OKF reserved file (¬ß7). No frontmatter. Newest entries first.
Append (prepend a new section) with today's date and what was regenerated:

```markdown
# Log

## 2026-06-29

**Regeneration** ‚Äî full `/anatomy` run; N areas indexed.
```

### Step 6 ‚Äî Update `ANATOMY.md`

Keep it to a single table. In refresh mode, update only the rows for touched areas.

```markdown
# Repository Anatomy

Navigable map of the codebase. Each area links to its wiki page; open that before grepping.

| Area | Purpose | Wiki |
|---|---|---|
| web | Next.js SPA ‚Äî graph explorer + spec UI | [web](docs/wiki/web.md) |
```

### Step 7 ‚Äî Render `viz.html` (full mode only)

After all writes, render the unified interactive graph from the whole `docs/` bundle
(wiki + specs + standards), excluding the generated viz file itself:

```bash
uv run .claude/scripts/okf_visualize.py docs/ -o docs/viz.html --exclude docs/viz.html
```

If `uv` is unavailable:

```bash
pip install --quiet pyyaml && python3 .claude/scripts/okf_visualize.py docs/ -o docs/viz.html --exclude docs/viz.html
```

Report the node and edge count from the render output. On success, tell the user:
`open docs/viz.html` to view the graph locally.

> `/anatomy` still **validates** `docs/wiki/` as its own sub-bundle, but the rendered
> graph spans the full `docs/` bundle.

### Step 8 ‚Äî Keep edits minimal and idempotent

Don't reflow unchanged sections. Don't touch files outside `ANATOMY.md`, `docs/wiki/`,
and `docs/viz.html`. Never fail a commit ‚Äî if you cannot complete, write nothing
and report briefly.

## Output

End with a one-line summary: which areas were refreshed, whether `viz.html` was rendered,
or `no tracked source files ‚Äî skipped`.

## OKF v0.1 quick reference

- **Bundle**: directory of `.md` files with YAML frontmatter.
- **Required**: `type` field in every non-reserved `.md` file.
- **Recommended**: `title`, `description`, `tags`, `timestamp`.
- **Reserved filenames**: `index.md` (directory listing), `log.md` (history). Neither is a concept.
- **Cross-links**: standard markdown links, bundle-relative. `[area](area.md)`.
- **Validate**: `uv run .claude/scripts/okf_validate.py docs/wiki [--strict]` (wiki sub-bundle)
- **Visualize**: `uv run .claude/scripts/okf_visualize.py docs/ -o docs/viz.html --exclude docs/viz.html`
