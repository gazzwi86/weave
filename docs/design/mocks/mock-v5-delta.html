Glossary

NEW · v1

Business terms and definitions with SKOS broader/narrower links. No M1 screen — terms weren't browsable at all.

Click-and-Collect
 Order online, collect in-store — reserves stock at a nominated branch.
broader: Fulfillment narrower: Locker Pickup
RMA
 Return Merchandise Authorisation — reference issued before a return is accepted.
broader: Returns
Stock Keeping Unit (SKU)
 Unique identifier for a distinct sellable product variant.
broader: Inventory narrower: Variant SKU
Goods Inward
 Receiving and checking-in of supplier deliveries at a store or DC.
broader: Inbound Logistics
Pricing Tier
 Customer-segment price band (retail / trade / contract).
broader: Pricing
Loss Prevention
 Measures to reduce shrinkage from theft, error or fraud.
broader: Store Ops

Improvement

    PREVIEW · post-v1
  

  
Store Ops Portal. Self-generated proposals for how this project could run better — perf, cost, reliability, code health. Distinct from Weave's own platform-level self-improvement (super-admin, analyses Weave's own logs/prompts) — same technique, different scope. Both post-v1.


  

    

      Cache Instances list queries
      Perf
    

    
Instance list re-queries the graph on every filter change — 40% of requests are repeat queries within 5 min. Add a 60s cache.

    

  

  

    

      Downgrade nightly SHACL revalidation to changed shapes only
      Cost
    

    
Full revalidation runs nightly regardless of change volume — 3 of the last 7 runs touched <2% of shapes. Scope to delta.

    

  

  

    

      Extract refund-calc retry logic into shared util
      Reliability
    

    
Retry-with-backoff duplicated in 3 tasks this milestone. Accepted — merged in TASK-014 follow-up.
