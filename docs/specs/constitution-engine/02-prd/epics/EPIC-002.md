# Epic: EPIC-002 - Instance Data Population

## Overview

**Phase:** Phase 1 — Validated-Mutation Spine (MVP)
**PRD Reference:** [prd.md](../prd.md#epic-2-instance-data-population)
**Status:** Backlog

## Description

Lets a business analyst populate the graph with real company entities — adding, editing, deleting,
browsing, and searching instances of the client's domain classes — via chat or guided forms, never
raw RDF. Edits use partial-update semantics so AI changes never wipe unrelated properties (such as
canvas layout). Bulk-populate via CSV (E2-S3) is carved out to Phase 4 and is not delivered here.

## Acceptance Criteria (Epic Level)

- [ ] An edit that omits a property (e.g. position, colour, domain) leaves that property **untouched**
      after commit — partial-update semantics (FR-034) hold across every instance mutation in this
      epic, whether authored via chat or form.
- [ ] Deleting an instance referenced by other entities surfaces a warning listing every affected
      reference and requires explicit confirmation before commit; declining aborts with the graph
      unchanged. Delete-own RBAC (a BA may delete only instances they authored) is enforced.
- [ ] The Instances / Data screen lists and text-searches instances grouped by class and **paginates**
      (default 50/page, tunable) rather than silently truncating — a class with more than one page of
      instances never loses rows.
- [ ] Every add/edit/delete routes through the single validated mutation entry point (CE-WRITE-1),
      is SHACL-checked against the target class's shapes, and carries a PROV-O record including the
      approving human identity and authoring-agent kind.

## Technical Notes

- Partial-update is grounded in the prototype's `update_node` `exclude_unset` behaviour — only named
  properties are retracted/asserted; all others are preserved. This is the load-bearing protection
  against AI edits wiping canvas layout.
- Class selection for a new instance must resolve against the **active** ontology (a client `System`
  subclass, etc.), not a hardcoded list — the screen is vocabulary-agnostic (FR-020/FR-021).
- The delete-referencing-warning flow (E2-S2) is the same flow that scopes delete-own RBAC in the
  PRD §6 matrix footnotes; reuse one referential-integrity check for both.
- Pagination default (50/page) is a PLAT-SETTINGS-1 tunable; never hardcode a silent cap.
