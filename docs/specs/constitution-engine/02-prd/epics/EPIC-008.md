# Epic: EPIC-008 - OWL Reasoning

## Overview

**Phase:** Phase 4 — Publish-Time Reasoning + Deferred Should-Haves (Post-MVP)

**Status:** Backlog

## Description

Adds publish-time OWL reasoning: a pre-publish consistency check that blocks inconsistent versions,
and per-version inference materialisation into immutable inferred named graphs. This phase is
decision-gated on OQ-01 (which reasoner ships, tractable at 500k triples) and is not on the MVP thin
loop; the MVP publish path runs without the heavy reasoner and is upgraded in place here.

## Acceptance Criteria (Epic Level)

- [ ] An OWL **consistency check runs before each publish**; an inconsistent draft (unsatisfiable
      classes, violated disjointness) blocks the publish with the affected classes + violated axioms,
      and a reasoner that is **unavailable** also blocks the publish — the system never publishes
      unchecked; the draft remains editable (FR-007).
- [ ] Inference materialises into a **per-version** inferred named graph (e.g.
      `weave:graph/v1.2.0/inferred`) at publish; a pinned read of v1.2 **never** sees inferences from
      v1.3, and inferred triples are **labelled as inferred** (not asserted) in query results and
      instance views (FR-028, FR-029).
- [ ] A reasoner that exceeds its budget (default 30 s, tunable per workspace) produces a
      reasoner-timeout error and **no partial inferred graph** is committed — partial materialisation
      is never observable.
- [ ] These checks compose with the earlier SHACL gate without overlap: `sh:Violation` blocks at
      **commit** (EPIC-006); OWL inconsistency blocks at **publish** (here) — the boundary is explicit
      and tested.

## Dependencies

- **Blocked by:** **OQ-01 resolution** (reasoner choice + tractability — a hard Phase 4 entry gate);
      EPIC-009 (the draft→published version lifecycle that reasoning hooks into at publish);
      EPIC-001 (the OWL structure reasoning operates over). Phase 3 gate must have passed.
- **Blocks:** GA (this is the last engine phase before GA per the roadmap); downstream consumers that
      want inference-labelled, consistency-checked published versions.

## Technical Notes

- The reasoner is **OQ-01-gated**: candidates include RDFLib-OWL / Owlready2 / Stardog / ELK; the
  fallback lever (PRD §10 Risks) is "consistency-check-only" if full materialisation is intractable at
  500k triples — the consistency check (E8-S2) is the Must-Have floor, materialisation (E8-S1) the
  Should-Have extension.
- Reasoning is **batch at publish**, deliberately distinct from the per-commit SHACL gate (validation/
  reasoning-timing decision) — do not move it to commit time.
- Per-version inferred graphs must be immutable like their published version (a v1.2 read is
  byte-deterministic); never mutate an inferred graph after publish.
- Standards: OWL 2 DL reasoning, named-graph versioning, inferred-triple labelling, PROV-O.
