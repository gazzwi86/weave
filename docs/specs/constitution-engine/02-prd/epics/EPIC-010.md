# Epic: EPIC-010 - Stable Read & Write Interfaces (Inter-engine Contract Hub)

## Overview

**Phase:** Phase 1 — Validated-Mutation Spine (MVP)

**Status:** Backlog

## Description

Exposes the two core contract-hub interfaces every downstream engine depends on: CE-READ-1, a
versioned, SELECT-only, `SERVICE`-blocked, paginated, tenant-scoped read interface; and CE-WRITE-1,
the single validated `POST /api/operations/apply` write entry point that clones, SHACL-validates, and
commits with a PROV-O activity. These are the stable, versioned public faces of the spine built in
EPIC-006 and EPIC-009.

## Acceptance Criteria (Epic Level)

- [ ] CE-READ-1: `GET /api/sparql?version=<iri|latest>&page=<n>` is **SELECT-only**, `SERVICE`-blocked,
      **paginated** (never silently truncates), and tenant-scoped; `GET /api/ontology/types|resource/{iri}|versions`
      serve registered kinds + client extensions, a single entity, and the version list — each accepts
      `?version=` defaulting to `latest`; a non-existent pinned version → 404.
- [ ] CE-WRITE-1: `POST /api/operations/apply` accepts `{operations, actor, target}`, applies on a
      **throwaway clone**, SHACL-validates, commits only if no `sh:Violation`, writes a PROV-O activity
      attributed to `actor`, and returns `201 {activity_iri, applied_count, version_iri}` or
      `422 {violations}`; a malformed `Op` → 400 without touching the store; duplicate-IRI create is
      reconciled (idempotency key supported).
- [ ] CE-WRITE-1 is the **only** mutation entry point — a CI/contract test confirms no second write
      path exists and that a batch with any `sh:Violation` commits **nothing** (all-or-nothing).
- [ ] All endpoints require a Cognito JWT / service principal and are tenant-scoped; an unscoped query
      that would cross tenant boundaries is rejected (no-JWT → 401; cross-tenant read → zero foreign
      triples).

## Technical Notes

- This epic **publishes** the spine as stable contracts; the validation/versioning behaviour itself
  lives in EPIC-006/EPIC-009 — keep one implementation, two public faces. Do not re-implement
  query-safety or clone-validation here.
- `Op ∈ add_node | update_node | add_edge | delete_node | delete_edge`; new nodes carry a local `ref`
  resolved to IRIs in-batch; dedup = case-insensitive `label`+`kind` reuse (CE-WRITE-1 §5 shape).
- CE-READ-1 query-safety is the same SELECT-only/`SERVICE`-blocked/paginated sanitizer shared with
  EPIC-007's editor (decision B3) — one guardrail.
- Both contracts are marked **stable** in §5; their exact OpenAPI shapes are fixed in the Phase-1 tech
  spec. Standards: OpenAPI 3.1, SPARQL 1.1 SELECT, SHACL, PROV-O, Cognito JWT.
