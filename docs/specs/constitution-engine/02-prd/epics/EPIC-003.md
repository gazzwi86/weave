# Epic: EPIC-003 - SKOS Controlled Vocabulary (Glossary)

## Overview

**Phase:** Phase 2 — Governed Content + Query + Contract Hub (MVP)
**PRD Reference:** [prd.md](../prd.md#epic-3-skos-controlled-vocabulary-glossary)
**Status:** Backlog
**Priority:** Must Have

## Description

Gives the organisation one agreed meaning per business term: a SKOS-based glossary where each term
is created with a preferred label, definition, synonyms, and broader/narrower links, searchable by
any user. A glossary term and its structural OWL class are the **same punned resource** (decision
B1), so vocabulary and structure can never drift out of sync.

## Acceptance Criteria (Epic Level)

- [ ] A single URI is simultaneously an `owl:Class` and a `skos:Concept` (B1) with **no** separate
      linking property (the legacy `weave:denotes` pattern is unsupported); the reconciliation query
      "show everything we know about X" returns both the OWL axioms and the SKOS annotations of that
      one URI.
- [ ] SHACL enforces exactly **one** `skos:prefLabel` per language and exactly **one**
      `skos:definition` per concept; a second prefLabel in the same language is rejected with HTTP
      422 — and this holds with validation running `inference='none'` (DL-completeness is not
      load-bearing).
- [ ] Glossary search matches on `skos:prefLabel`, `skos:altLabel`, and `skos:definition`, and each
      result shows the resource's OWL role (because it is punned); a no-match search returns an
      empty-state with a "create this term" affordance, never an error.

## Technical Notes

- Punning (B1) is grounded in the obpm `mi-glossary.ttl` prototype where the same URI is both an
  `owl:Class` and a `skos:Concept`. Document the punning in the data model; do not add a cross-notation
  linking property.
- Validation runs with `inference='none'` precisely because punning would otherwise make DL-completeness
  load-bearing — keep the SHACL engine in closed-world mode (FR-022).
- Standards: SKOS vocabulary (`prefLabel`/`altLabel`/`definition`/`broader`/`narrower`), OWL 2 DL,
  SHACL cardinality constraints.
- The "create this term" empty-state affordance feeds the same CE-WRITE-1 mutation pipeline as
  E3-S1 — no separate creation path.
