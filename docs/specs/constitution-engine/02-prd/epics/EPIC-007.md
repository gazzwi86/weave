## Description

Two query surfaces over the graph plus the agent-grounding read pattern: a plain-English NL question
box that generates and executes a SPARQL SELECT, a raw SPARQL editor restricted to SELECT-only,
`SERVICE`-blocked, paginated queries (decision B3), and built-in **agent-authority SELECTs** that let
a caller ask what an agent may do — on which systems/data/process, and who to contact or escalate to —
answered from the modelled BPMO relationships over CE-READ-1 (no new contract). Both query surfaces
degrade gracefully when the AI is offline. Server-side saved/shared queries (E7-S3) are carved out to
the deferred-should-haves phase and are not delivered here.

## Acceptance Criteria (Epic Level)

- [ ] The query surface is **SPARQL 1.1 SELECT-only**: UPDATE/INSERT/DELETE are rejected and the
      `SERVICE` keyword is blocked (SSRF) **before execution** — a non-SELECT or `SERVICE`-bearing
      query never reaches the store, whether typed in the editor or generated by the NL path.
- [ ] Results are **paginated** with no silent row cap, on both the NL path and the raw editor.
- [ ] The NL path shows the generated SPARQL (collapsed, expandable) and, when it cannot construct a
      valid query, asks a clarifying question rather than hallucinating results.
- [ ] With the AI provider unavailable, the NL surface returns HTTP 503 with a clear message **while
      the raw SELECT-only editor stays fully functional** — the user can still query without AI.
- [ ] The editor pre-loads the standard prefixes (rdf, rdfs, owl, skos, prov, dcterms, xsd, weave,
      res) and renders SELECT bindings as a table with syntax highlighting + prefix auto-complete.
- [ ] **Agent-grounding (E7-S4, FR-036):** built-in agent-authority SELECTs answer "what may an agent
      do, on which systems/data/process, who to escalate to" from the modelled
      `governedBy`/`performedBy`/`accesses` links over CE-READ-1 — **no new contract is minted**; where
      the graph does not state a permission the answer defaults to **deny / route-to-human** (default,
      tunable) and an explicit deny overrides any inferred authority; a missing required link (e.g. a
      process with no `performedBy`) returns an explicit **coverage-gap row**, never an empty result
      that could read as "permitted".

## Technical Notes

- The SELECT-only / `SERVICE`-blocked / paginated sanitizer is the **same** guardrail as CE-READ-1
  (decision B3, grounded in the prototype sanitizer `store.py:581-603`) — share one implementation;
  do not fork query-safety logic between the editor and the contract.
- CONSTRUCT/ASK/DESCRIBE are explicitly deferred (OQ-10) — do not promise them in v1.
- The NL→SPARQL path must never bypass the SELECT-only gate: the generated query is sanitized exactly
  like a hand-typed one before execution.
- Agent-grounding (E7-S4) is a **read-side** pattern, not a new contract: the authority answer is
  computed entirely from modelled BPMO relationships over CE-READ-1. The deny/route-to-human default
  and the coverage-gap-row behaviour are the load-bearing safety properties — an absent permission must
  never resolve to "permitted". Pattern grounded in the obpm `mi-agent-model.ttl`
  authority/permission/HITL model.
- Standards: SPARQL 1.1 SELECT, SSRF prevention (`SERVICE` block), pagination contract (no silent cap).
