# Epic: EPIC-004 - Brand & Voice Standards

## Overview

**Phase:** Phase 2 — Governed Content + Query + Contract Hub (MVP)
**PRD Reference:** [prd.md](../prd.md#epic-4-brand--voice-standards)
**Status:** Backlog
**Priority:** Must Have

## Description

Gives the brand / marketing owner a single authoritative, versioned, provenance-stamped home for
brand styleguides and tone-of-voice rules, stored as governed RDF individuals. Projects them through
CE-BRAND-1 as flattened design-token JSON and machine-evaluable VoiceRules so the Build Engine can
generate compliant-by-construction artefacts without parsing RDF.

## Acceptance Criteria (Epic Level)

- [ ] `GET /api/brand/tokens` returns flattened design-token JSON (colour, type scale, spacing,
      radii…) and `GET /api/brand/voice-rules` returns machine-evaluable VoiceRules — both consumable
      by Build **without parsing RDF** (CE-BRAND-1 contract test passes at the §5 shape).
- [ ] A brand individual that fails its SHACL shape is blocked at commit (HTTP 422) and **never**
      appears in the token projection or the voice-rule projection — the projection can only ever
      expose validated content.
- [ ] With the AI provider unavailable, the owner can still add brand individuals and VoiceRules
      through guided forms; the styleguide-extraction surface returns 503 without blocking manual
      entry.
- [ ] Every brand/voice change is versioned and carries a PROV-O stamp (routed through CE-WRITE-1).

## Dependencies

- **Blocked by:** EPIC-006 (SHACL validation gating the projection); EPIC-009 (versioning + PROV-O);

## Technical Notes

- CE-BRAND-1 is a **projection** over RDF brand individuals, not a separate store — the RDF is the
  source of truth; tokens/voice-rules are derived views. Re-derive on read or on commit; never let
  the projection diverge from the validated graph.
- VoiceRules carry a human label **and** a machine-evaluable assertion (each a checkable assertion);
  Build evaluates generated text against them, so the assertion shape must be stable.
- E4-S2 (VoiceRule **extraction** from a pasted styleguide via AI) is Should-Have; manual form entry
  is the Must-Have floor and the 503-fallback path.
- Standards: RDF individuals, SHACL shapes, PROV-O, design-token JSON projection (CE-BRAND-1).
