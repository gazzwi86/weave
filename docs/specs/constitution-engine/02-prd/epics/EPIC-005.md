## Description

Lets a compliance / risk officer describe a regulatory obligation in plain English and have the AI
generate a **tenant-scoped** SHACL shape that is then enforced on every future edit, plus browse all
modelled rules with live violation coverage. Scheduled self-audit gap queries (E5-S2) are carved out
to Phase 4 and are not delivered here.

## Acceptance Criteria (Epic Level)

- [ ] A tenant's custom SHACL shape is stored in **this tenant's** shapes graph (never global) and
      MUST NOT affect any other tenant — the mandatory **cross-tenant shape-leak test** (tenant-A
      shape, tenant-B commit unaffected) passes (FR-025, NFR Isolation).
- [ ] A shape change **invalidates validation caches across all workers/instances** (external
      invalidation, not process-local) so the new rule takes effect on the very next commit on any
      worker.
- [ ] The Rules & Policies screen lists every SHACL shape with target class, constraint summary, and
      severity (including `sh:Info`), and for each shape shows which entities are currently in
      violation; if validation has not yet run for the current draft it shows a "validation pending"
      state, never stale or empty coverage.
- [ ] Each generated shape carries its own PROV-O provenance and is reviewed by the officer before it
      is added (routed through CE-WRITE-1).

## Technical Notes

- The per-tenant shapes graph is the heart of the cross-tenant security property — it shares the
  same isolation mechanism (store-per-tenant OR named-graph + query-rewriting, OQ-04) as instance
  data; the shape-leak test is a required security-review sub-gate this phase.
- Cache invalidation must be **external** (shared, e.g. Redis/ElastiCache) so a shape edited on one
  worker is honoured by all — a process-local cache would silently apply stale rules.
- PII / sensitive handling uses **SHACL + data-classification properties**, NOT ODRL in v1 (OQ-09);
  do not introduce an ODRL dependency.
- Severity handling aligns with EPIC-006: `sh:Violation` blocks; `sh:Warning`/`sh:Info` are advisory
  and surfaced, never blocking.
