## Implementation

### Pseudocode

**TaskBrief schema (Pydantic v2):**

```python
class EarsAC(BaseModel):
    id: str                   # "AC-1", "AC-2", …
    criterion: str            # must start with "WHEN " — validated by @field_validator
    test_mapping: str         # named test stub

class DepChain(BaseModel):
    blocked_by: list[str]     # TASK-NNN or empty list
    unlocks: list[str]

class CostEstimate(BaseModel):
    complexity: Literal["S", "M", "L", "XL"]
    estimated_tokens_input_k: int
    estimated_tokens_output_k: int
    estimated_cost_usd: float

class TaskBrief(BaseModel):
    schema_version: Literal["1.0"]
    task_id: str
    project_iri: str
    title: str
    user_story: str           # "As a … / I want … / So that …"
    acceptance_criteria: list[EarsAC]   # min 1; each criterion validated EARS
    ac_to_test_map: list[dict]          # every AC-id must appear
    dor_checklist: list[str]            # all items present
    dod_checklist: list[str]
    dep_chain: DepChain
    cost_estimate: CostEstimate
    design_tokens: dict | None = None   # reserved for M2 CE-BRAND-1
    generated_at: datetime
```

**Architect agent invocation (light pseudocode — Opus advisor owns deep implementation):**

```
function generate_task_brief(jwt, project_iri, task_description, dep_summaries=[]):
  claims = cognito.verify(jwt)       # → 401
  if not project_iri: return 422 with {"error": "validation_error", "field": "project_iri"}

  # Ground brief in CE graph
  bpmo_context = ce_read_client.get(f"/api/ontology/resource/{project_iri}")
                                     # raises on non-2xx → 503

  # Architect agent (claude-opus-4-8): draft brief from task_description + bpmo_context
  raw_brief = architect_agent.run(
    model="claude-opus-4-8",
    prompt=build_architect_prompt(task_description, bpmo_context, dep_summaries),
    tools=[ce_read_tool]
  )  # routing miss → halt + PLAT-AUDIT-1 event

  # Validate schema (claude-haiku-4-5 structural pass, then Pydantic)
  validated = TaskBrief.model_validate(raw_brief)
  # Pydantic raises ValidationError → 422 with missing_fields

  # Persist to RLS-isolated store
  task_id = generate_task_id(project_iri)  # deterministic UUID v5
  brief_iri = f"urn:weave:brief:{task_id}"
  aurora.execute(
    "INSERT INTO task_briefs (task_id, project_iri, tenant_id, brief_iri, content, schema_version)"
    " VALUES (:tid, :piri, :tenant, :biri, :content, '1.0')",
    tid=task_id, piri=project_iri, tenant=claims["custom:tenant_id"],
    biri=brief_iri, content=validated.model_dump_json()
  )

  emit_audit_event("brief_generated", actor=claims.sub, target=brief_iri)  # PLAT-AUDIT-1

  return 201 with {"task_id": task_id, "brief_iri": brief_iri, "stored_at": "<iso8601>"}
```

### API Contracts

**`POST /api/projects/{project_iri}/briefs`**

Request body:

```json
{
  "task_description": "string — natural-language task description (required)",
  "dep_summaries": ["string — predecessor task_id, used to load handoff summaries (optional)"]
}
```

Response `201`:

```json
{
  "task_id": "string — UUID v5 deterministic task identifier",
  "brief_iri": "string — urn:weave:brief:{task_id}",
  "stored_at": "string — ISO 8601 UTC"
}
```

Error responses:

| Status | Condition | Body |
|---|---|---|
| 401 | Missing or invalid Cognito JWT | `{"error": "unauthorised"}` + `Www-Authenticate: Bearer` |
| 422 | Generated brief fails schema validation | `{"error": "brief_invalid", "missing_fields": ["<field>", ...]}` |
| 422 | `task_description` absent or empty | `{"error": "validation_error", "field": "task_description"}` |
| 503 | `CE-READ-1` unreachable | `{"error": "ce_read_unavailable"}` |

**`GET /api/projects/{project_iri}/briefs/{task_id}`**

Response `200`:

```json
{
  "task_id": "string",
  "brief_iri": "string",
  "schema_version": "string — e.g. \"1.0\"",
  "content": "object — full TaskBrief document"
}
```

Error responses:

| Status | Condition | Body |
|---|---|---|
| 401 | Missing or invalid JWT | `{"error": "unauthorised"}` |
| 404 | Brief not found or belongs to another tenant | `{"error": "not_found"}` |

## Implementation Hints

- The `@field_validator` on `EarsAC.criterion` should be `mode="before"` so it fires on raw
  string input: `assert value.startswith("WHEN "), "AC criterion must begin with 'WHEN'"`.
- The Architect agent uses `claude-opus-4-8`; wire it via the Anthropic Agent SDK's
  `Agent(model="claude-opus-4-8", ...)`. Never pass model ID as a runtime variable — use a
  named constant that fails fast at startup if the ID is not in the allowed set.
- The 200-node context cap (OQ-11) applies when loading BPMO context from CE-READ-1; implement
  a relevance filter (top-N nodes by relationship proximity to `project_iri`) before prompt
  assembly. This is the M1 mitigation; FR-051 investigator pattern lands in M2.
- `UUID5` for `task_id` uses the project IRI + task description as the name — deterministic
  so retries produce the same ID and the `INSERT` can be idempotent on conflict.
- Do not store raw LLM output directly; always run `TaskBrief.model_validate()` first and
  store only the validated JSON. This keeps the store clean regardless of model quirks.
