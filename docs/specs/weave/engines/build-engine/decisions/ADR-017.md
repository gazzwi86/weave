# ADR-017: Phase-gate ceremony + spec-coverage audit design (no requirement registry)

## Status
Accepted (TASK-008, EPIC-012). Coordinator-authored to resolve the docstring references in
`ceremony.py` / `spec_coverage.py`, recording the four design calls QA confirmed coherent.

## Context
TASK-008 builds the phase-gate ceremony (FR-052) and spec-coverage audit (FR-053), but the codebase
has **no FR/NFR requirement registry** to audit coverage against, and the ceremony must be
human-gated without any auto-approve path.

## Decisions
1. **Ceremony as a fixed-order data-table step loop** (`security → mutation → qa_full →
   coverage_audit → summary`), one `record_gate` row per step; any step exception fails the whole
   ceremony **closed**. Not a workflow engine.
2. **Spec-coverage model:** audit coverage against **every AC in every in-scope task brief's
   `acceptance_criteria` + `ac_to_test_map`** (the brief's own prescribed fallback for the absent
   registry) + test-tree presence. Ambiguous (no mapping row) → **MISSING**; halt below 90% delivered
   or on any MISSING. Defaults DOWN (stricter), never up.
3. **`approve_blocked` ≠ ceremony-failed:** a CRITICAL security / sub-threshold mutation finding sets
   `approve_blocked=True` (blocks the Approve action) but does NOT halt the loop — Amend/Reject stay
   live. Distinct states.
4. **Acting-principals proxy:** derived from `audit_entries` (every principal that produced a commit
   or gate row within the phase), used to reject self-approval. HITL approval is entirely
   caller-driven — no internal code path supplies a hardcoded `"approve"`; fail-closed on audit outage.

## Consequences
- Sound + faithful to the brief; QA-verified (all 8 ACs, 100% coverage, no auto-approve path).
- **Open follow-up (WARN):** task-level `priority` (Must/Should) is not yet filtered — every AC is
  treated as Must. Moot today (no production caller wires `QAProject` from real task-brief files yet);
  the future wiring task should honour `priority`. Safe direction (over-cautious halt, never false pass).
