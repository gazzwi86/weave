# ADR-013: Authority Extension deferred post-v1; M2 authority() is base-links only

## Status

Accepted — 2026-07-08 (adversarial red-team remediation). Supersedes the **M2 build phasing**
stated in program [ADR-002 (authority extension)](../../../decisions/ADR-002-authority-extension.md);
that ADR's *direction* (ODRL 2.2 / SKOS / SHACL / PROV-O, single source of truth) is unchanged
and remains accepted. Marks **OQ-AUTH-1 = deferred post-v1**.

## Context

The red-team found the M2 `authority()` brief (v1/tasks/TASK-010) unbuildable as written: the
base 13-kind BPMO cannot express `Permission`, `authorityLevel`, explicit deny, or HITL
triggers — those live in the ODRL Authority Extension, which has no M2 producer and no M2
consumer (Events & Actions is post-v1). Contracts.md (CE-READ-1) already pins the honest
degrade: "Without extension populated, `authority()` degrades to `coverage_gap` + `deny` —
never returns implicit allow", with response convention
`{ rows, decision: "permit"|"deny"|"coverage-gap" }`.

## Decision

1. **M2 ships ONLY the base-links surface:** deny-default `authority()` (response per CE-READ-1
   convention; `decision` is never `"permit"` in M2), `coverage_gap(kind, required_links[])`
   (default invocation `(Process, [performedBy, governedBy])`, rows
   `{ entity_iri, missing_link }`), and base `escalation(process)` (Actor(s) to contact — no
   deadline evaluation).
2. **The full Authority Extension is a POST-V1 backlog item:** ODRL 2.2 vocabulary
   (`odrl:Policy/Permission/Prohibition/Duty`), `Permission` resolution via
   `Actor —holdsRole→ Role —assignee→ Permission`, `authorityLevel` SKOS ordered scheme,
   explicit-deny override, `HITLTrigger` (`escalatesTo` / `escalationDeadline` /
   `triggeredByStep`), and the `dataClassification`-gated Duties. Ship-canonical vs
   document-client-pattern (**OQ-AUTH-1**) is decided when the item is picked up — deferred
   post-v1.
3. No CE artifact may cite "ADR-002" bare for authority: CE-local ADR-002 is the
   provenance/version-lifecycle decision. Authority citations use "program ADR-002" (vocabulary
   direction) and this ADR (phasing/descope).

## Consequences

- v1 TASK-010 becomes buildable: every AC resolves from links the base BPMO can express.
- Agents never receive a `permit` before the extension exists — fail-closed is preserved and
  strengthened (no code path can return permit in M2).
- Explicit-deny-overrides and escalation deadlines are not testable until post-v1; briefs and
  exit criteria must not claim them earlier.
- Program ADR-002's "extension module ships M2" sentence is stale — **escalated to the program
  spec owner** (file is outside CE ownership).

## Alternatives considered

- **Build the ODRL extension in M2 anyway** — rejected: no consumer until Events (post-v1),
  violates the CE-FUNCTION-1 precedent (build alongside the first consumer) and the red-team
  finding stands (briefs oversold unbuildable ACs).
- **Drop authority() from M2 entirely** — rejected: the deny-default + coverage-gap surface is
  the load-bearing safety floor contracts.md already promises consumers.
