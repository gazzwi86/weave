## modules/

Shared module code; environments vary only via tfvars.

- `cognito/` — user pool + client; token validity at AWS floors per ADR-001 (access/ID 300s, refresh 3600s)
- `secrets/` — small dev Secrets Manager prefix `weave-dev`
- `s3_state/` — TF state bucket, versioned + SSE
- `dynamo_lock/` — TF lock table, PAY_PER_REQUEST
- `vpc/`, `aurora_pg/` (Serverless v2), `elasticache/` (Redis 7.1), `s3_assets/`, `s3_spa/`, `cloudfront/` — prod-gated via `count = var.deploy_prod_stack ? 1 : 0`

## environments/

- `dev/` — the single root module (`main.tf`, `providers.tf`, `variables.tf`, backend "s3" block, `dev.tfvars`); offline plan/validate via git-ignored local-backend `override.tf` (tests in `packages/backend/tests/integration/test_terraform_plan.py`)
- `staging/`, `prod/` — tfvars-only placeholders (copy-later; deliberate YAGNI, flagged in the epic PR)

## Deploy path

CI `deploy-essential-dev` job (`.github/workflows/ci.yml`) — OIDC role assumption (`vars.DEPLOY_ROLE_ARN`, no stored keys), `terraform plan` full stack, `apply -var deploy_prod_stack=false`. No-ops until a human bootstraps the deploy role (trust-policy shape in `infra/terraform/README.md`).
